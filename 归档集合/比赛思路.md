# 比赛思路

```人话
1. pad连接机器人，使用pad的app获取机器人IP，再使用VNC远程登录或者直接外接显示器直接使用。pad连接应该主要是舵机校准。
2. 运动控制即舵机控制，传感器模块操作。应该直接调用Restful API就可以了，需要查看API可以在机器人内部输入指定IP查看；http://127.0.0.1:9090/v1/ui；举手前进等；
3. 深度学习应该只是使用keras，视频里的是手势识别。训练周期不会太长(10周期之内)；但是可能需要部署到机器人身上，所以需要VNC来传输文件。其中编写代码使用pycharm，关于pycharm可以打开它的scientific mode 来查看函数相关文档。如果要自己编写训练和验证代码的话就需要总结一套简单好记的写法，可能也会有猫狗识别，人脸识别、手势识别（视频里面提到的），一般是有关图像的CNN。
4. 机器人进行通讯场景模拟，与其他设备通信。GPIO编程/Web编程；我想的是让机器人和同网段PC通信；PC发送网络请求控制机器人的GPIO，达到通信与物联网要求；可能需要flask来实现PC服务端，机器人客户端；
5. 场景设计综合以上4个任务的各功能模块且需要包含6个方面：运动、传感（API）、视觉识别（CV）、语音识别（API）、物联网（GPIO）、网络通信（Web）；

```

# 比赛准备

```
1. 正式比赛前一个月会发布样题，这个需要留意。
2. 熟悉任务操作，其中API调用要流程化，要会在127.0.0.1:9090/v1/ui下查机器人API，应该掌握Restful API这最新API就可以。
3. 根据比赛的硬件环境推测比赛题目要求，其中传感器触摸、红外、超声、温湿度要掌握其调用API和基础数据应用；USB台灯和警报灯可能是用于GPIO的，掌握端口接线和编程实现。还有颜色小球视频里面有追踪颜色小球案例，也可能颜色识别，应该考察的是opencv和picamera库的使用，还用到了多线程处理。
4. 关于机器人应用部分需要重点掌握应该是拍照，视频流处理等有关还是视觉方面的库。
5. 考虑到PC端装有pycharm的可能没有安装相关keras的库，所以函数提示和函数帮助可能没有。所以还是得自己掌握keras的数据预处理、设计神经网络以CNN为主，训练模型（短周期，有效果就行），部署以及配合API应用。
6. 总分相同按任务5场景设计得分排名；

```
